Classes in Ruby

June 21, 2015

Classes are Objects

Classes are objects. They're the only kind of object that has the power to spawn new objects(instances). When you create a class, you can send messages to it, add methods to it, pass it around to other objects as a method argument, and generally do anything to it you would do to another object. Take a look at this example:
Here I made a simple class called 'Car'. All class names begin with a capital letter. On the line below is the attribute accessor. This allows you to access the instance variables (:make, :model, and :year) from outside the class. We'll dive into that at another time. Next, we have the initialize method that is taking in some arguments. This method is executed when an instance of the class is created. Line 23 shows exactly how we create an instance of the class. I stored the new instance of 'Car' in the variable 'car1'. The initialize method is looking for three arguments to be entered. This is the ("BMW", "Z4", 2012). If it doesn't get 3 arguments, the program will raise an Argument Error.
Inside the initialize method are the assignments of the instance variables. These variables always start with the '@' symbol. These don't necessarily need to have the same names as the arguments, but it is standard procedure to use the same names to avoid confusion. Without the '@' symbol, you wouldn't have access to the values stored in them outside of the initialize method. Now that the instance variables are created, let's see how we use them throughout the class.
I created an instance method called 'lease_end' that returns a string with the instance values set in the initialize method. Then, I created another method called 'show' that creates a hash for the make and model of the car in each instance.
After creating a new instance of Car and storing in the variable car1, I called the show and lease_end methods on it. Then, I created another instance of Car and stored it in car2. I called the same methods on it and here is the output:
Classes give you the ability to create an instance that has all the methods established in that class. This Car class inherits from it's parent class, Object. This gives Car all of the methods accessible to Object. Ultimately, all objects in Ruby belong to the BasicObject superclass, which is the parent of Object. More on that next time!
