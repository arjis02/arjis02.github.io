Classes Vs. Modules

June 28, 2015

Creating a class in Ruby lets you group methods into convenient bundles, so that you can quickly create many objects that behave essentially the same way. Everything in Ruby belongs to a class. You can see this just by going into IRB and calling #class on any object.
As you can see in the picture above, classes have superclasses, also called parent classes. Each class within a class inherits the methods defined in its parent class. Every object you create is an instance of a class.
Another Ruby construct that's closely related to classes are modules. They encourage "modular" design. It breaks large components into smaller ones and lets you mix and match object behaviors. Like classes, modules are bundles of methods and constants. However, modules do not have instances. You can include a module into a particular class to add its functionality. Modules are, actually, a more basic structure. As you can see in IRB, the superclass of Class is Module.
Writing a module is similar to writing a class, except you start your definition with the 'module' keyword instead of the 'class' keyword. When you write a class, you would then create an instance of a class. As I said before, modules do not have instances. Instead, you would add a module into a class.
