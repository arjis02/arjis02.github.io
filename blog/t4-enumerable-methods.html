Enumerable#cycle

June 11, 2015

What is cycle?

Remember the Enumerable#each method? It would iterate through arrays and hashes and call a block for each element of enumerable. Well the cycle method is similar, except you can designate how many times you want to iterate through the enumerable. The other difference is that cycle returns nil once the loop has finished. See the example below:

a = ["yabba", "dabba", "doo"]

a.cycle{ |x| puts x } # prints yabba, dabba, doo, yabba, dabba, doo,.. forever

a.cycle(2){ |x| puts x } # prints yabba, dabba, doo, yabba, dabba, doo

In each instance, nil is returned. The '2' in the second example is the amount of times you want to call the block for each element. If you do not designate a number, it will repeat forever. If a non-positive number or zero is given, it simply returns nil.
Although the method ultimately returns nil, you can still use a block to perform some useful tasks. Consider the example below:

a = ["yabba", "dabba", "doo"]

b = []

array.cycle(3){ |x| b << x }

print b

["yabba", "dabba", "doo","yabba", "dabba", "doo", "yabba", "dabba", "doo"]=>nil

This can be used with hashes, as well.

c = [:h => 1, :i => 2, :j => 3]

array.cycle(2){ |k, v| puts "#{k} #{v+1}" }

If no block is given, an enumerator is returned instead. Practice makes perfect! Try it out.
