Classes are Objects

Classes are objects. They're the only kind of object that has the power to spawn new objects(instances). When you create a class, you can send messages to it, add methods to it, pass it around to other objects as a method argument, and generally do anything to it you would do to another object. Take a look at this example:
Here I made a simple class called 'Car'. All class names begin with a capital letter. On the line below is the attribute accessor. This allows you to access the instance variables (:make, :model, and :year) from outside the class. We'll dive into that at another time. Next, we have the initialize method that is taking in some arguments. This method is executed when an instance of the class is created. Line 23 shows exactly how we create an instance of the class. I stored the new instance of 'Car' in the variable 'car1'. The initialize method is looking for three arguments to be entered. This is the ("BMW", "Z4", 2012). If it doesn't get 3 arguments, the program will raise an Argument Error.
Inside the initialize method are the assignments of the instance variables. These variables always start with the '@' symbol. These don't necessarily need to have the same names as the arguments, but it is standard procedure to use the same names to avoid confusion. Without the '@' symbol, you wouldn't have access to the values stored in them outside of the initialize method.
